# 02_rtk_reverse_nav_pause_relay_initial_problematic

## 定位

这是最早加入“每隔固定距离停车 + USB 继电器闭合”的版本，也是暴露问题最多的版本。它不建议作为当前上车版本，只用于问题追溯和对比。

## 做了什么

- 在基础倒车控制上加入分段停车逻辑。
- 到达固定距离后停车保持。
- 停车阶段触发 USB 继电器闭合一段时间，用于喷洒动作。

## 为什么后来要改

这个版本的核心问题不是继电器本身，而是停车状态机破坏了连续控制：

1. 停车时输出 `linear=0, angular=0`，底盘/转向状态被打断；
2. 恢复导航后重新进入控制，容易出现大幅打轮；
3. 如果 `next_pause_progress` 使用固定累加而不是基于当前进度更新，可能刚起步又进入下一次停车；
4. 当停车间隔为 0.30 m 且起停时间较长时，车辆几乎一直处于“停—起—停—起”，没有足够连续距离修正横差。

## 工程效果

该版本解释了为什么“原本能稳定长距离倒车”的控制包，加入分段停车后会出现车轮大幅左右打、横差累计、甚至停车恢复时像原地自转。
